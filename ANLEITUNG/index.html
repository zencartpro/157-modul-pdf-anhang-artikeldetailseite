Sinn und Zweck dieses Moduls

Mit dieser Erweiterung kann zu einem Artikel über die normale Artikelbearbeitung in der Zen Cart Administration ein pdf Anhang hochgeladen werden.
Der Anhang kann mit einem Beschreibungstext versehen werden und wird dann auf der Artikeldetailseite unterhalb der Artikelbeschreibung für den Besucher zum Download angezeigt.
Nützlich z.B. wenn zu Artikeln im Shop ein pdf Datenblatt zum Download abgeboten werden soll.

Installation

Voraussetzungen


Dieses Modul ist nur für Zen Cart 1.5.7h deutsch geeignet
Stellen Sie sicher, dass Sie Ihren Shop auf 1.5.7h aktualisiert haben, bevor Sie dieses Modul installieren.
Die mitgelieferten geänderten Dateien sind auf dem Stand von 1.5.7h!

WICHTIG:

Für den Einbau dieses Moduls sollten Sie über folgende Werkzeuge verfügen:

1) Ein guter Texteditor.
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Meine Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

2) Ein Tool zum Vergleichen von Dateien
Bei der Installation dieses Moduls ist es erforderlich, dass Sie den Inhalt einiger Ihrer bestehenden Zen Cart Dateien mit dem Inhalt der neuen Moduldateien vergleichen und die Änderungen zusammenführen ("mergen").
Meine Empfehlung: BeyondCompare (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist das kostenlose Programm WinMerge


Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

WICHTIG
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

Die Installation erfolgt in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Im Ordner SQL die install.sql mit einem Texteditor öffnen und den Inhalt ins Eingabefeld kopieren unter Admin > Tools > SQL Patches installieren
Absenden
Dadurch werden die nötigen Datenbankeinträge vorgenommen.
Möglicherweise wird ein völlig harmloses Debug Log dabei erzeugt, das einfach löschen.

2)
Folgende Datei aus dem Shopverzeichnis herunterladen:
includes/configure.php

Mit einem Texteditor öffnen und folgenden Eintrag hinzufügen (z.B nach dem Eintrag define('DIR_FS_CATALOG', '/abcd/');)

define('DIR_WS_PDF_ATTACHMENTS', 'pdf-attachments/');

Geänderte includes/configure.php wieder hochladen. Diese Datei ist normalerweise schreibgeschützt, daher vor dem Hochladen der bestehenden Datei Schreibrechte geben (chmod 777).
Nach dem Hochladen wieder den Schreibschutz setzen (chmod 444)

3)
Folgende Datei aus dem Shopverzeichnis herunterladen:
DEINADMIN/includes/configure.php
DEINADMIN steht dabei für den Namen Ihres Adminverzeichnisses

Mit einem Texteditor öffnen und folgenden Eintrag hinzufügen (z.B nach dem Eintrag define('DIR_FS_CATALOG', '/abcd/');)

define('DIR_FS_CATALOG_PDF_ATTACHMENTS', 'xxx/xxx/xxx/xxx/pdf-attachments/');

Gemeint ist hier der vollständige Pfad zum Verzeichnis pdf-attachments im Shopverzeichnis

Den vollständigen Pfad zum Shopverzeichnis sehen Sie im Eintrag DIR_FS_CATALOG darüber, z.B.:
define('DIR_FS_CATALOG', '/var/html/user12345/shop/');

Für die neue Definition DIR_FS_CATALOG_PDF_ATTACHMENTS verwenden Sie exakt denselben Pfad und hängen einfach noch pdf-attachments/ an, z.B.

define('DIR_FS_CATALOG_PDF_ATTACHMENTS', '/var/html/user12345/shop/pdf-attachments/');

Geänderte DEINADMIN/includes/configure.php wieder hochladen. Diese Datei ist normalerweise schreibgeschützt, daher vor dem Hochladen der bestehenden Datei Schreibrechte geben (chmod 777).
Nach dem Hochladen wieder den Schreibschutz setzen (chmod 444)

4)
Im Ordner NEUE DATEIEN den Ordner DEINADMIN auf den Namen Ihres admin Verzeichnisses umbenennen.
Dann in der vorgegebenen Struktur ins Shopverzeichnis hochladen, dadurch werden keine bestehenden Dateien überschrieben.
Dem Ordner pdf-attachments nun Schreibrechte geben (chmod 777)

5)
Im Ordner GEAENDERTE DATEIEN den Ordner namens DEINTEMPLATE auf das im Shop verwendete Template umbennen (z.B. classic, ...).
Falls Sie im Shop das Template responsive classic verwenden, dann verwenden Sie den Ordner DEINTEMPLATE_RESPONSIVE

Den Ordner DEINADMIN auf den Namen Ihres admin Verzeichnisses umbenennen.
Wenn Sie Zen Cart 1.5.7h gerade frisch installiert haben und noch keinerlei Änderungen an den Dateien vorgenommen haben, können Sie nun alle Dateien/Ordner aus dem Ordner GEAENDERTE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen. Dabei werden dann etliche Dateien überschrieben.
Wenn Sie Zen Cart 1.5.7h schon länger im Einsatz haben und schon einmal Änderungen an Dateien vorgenommen oder andere Module eingebaut haben, dann laden Sie die Dateien KEINESFALLS einfach so hoch.
Vergleichen Sie stattdessen alle Dateien im Ordner GEAENDERTE DATEIEN mit den entsprechenden Dateien in Ihrem Shop und nehmen Sie die Änderungen manuell per WinMerge oder BeyondCompare vor.
Dann spielen Sie die geänderten Dateien in der gezeigten Struktur ein.

6)
In der Zen Cart Administration steht nun beim Bearbeiten eines Artikels eine neue Funktionalität für das Hochladen eines pdfs zum Artikel zur Verfügung. Sie funktioniert analog zum Hochladen eines Artikelbildes.
Tragen Sie im Feld Name für pdf Anhang einen Beschreibungstext ein, der später auf der Artikeldetailseite angezeigt werden wird. Wenn Sie im Shop mehrere Sprachen aktiv haben stehen Eingabefelder für alle Sprachen zur Verfügung (analog zum Artikelnamen).
Wählen Sie bei Datei für pdf Anhang mit Durchsuchen ein pdf von Ihrer Festplatte aus und aktualisieren Sie den Artikel.

Das pdf wird standardmäßig in den Ordner pdf-attachments hochgeladen.

Genau wie bei den Artikelbildern könnte auch hier mit Unterordnern gearbeitet werden (z.B. eigene Unterordner für die pdfs der verschiedenen Hersteller). Wenn Sie mit Unterordnern arbeiten wollen, dann legen Sie im Ordner pdf-attachments entsprechende Unterordner an. Achten Sie darauf, auch diesen Unterordnern Schreibrechte (chmod 777) zu geben. Sobald Unterordner vorhanden sind werden diese im Dropdown Uploadverzeichnis auswählbar sein.

Auf der Artikeldetailseite erscheint - falls ein pdf hinterlegt wurde - unterhalb der Artikelbeschreibung Ihr eingegebener Beschreibungstext und ein pdf Symbol zum Download der Datei.

Unter Webshop > Artikeltypen > Product General > Layouteinstellungen kann die Anzeige auf der Artikeldetailseite komplett deaktiviert werden.

Hinweis zu Dateitypen:
Für den Dateiupload sind standardmäßig nur pdf erlaubt. Wenn das ganze auch für andere Dateitypen genutzt werden soll, die Dateitypen in folgender Datei entsprechend ändern:
DEINADMIN/includes/modules/new_product_preview.php
$products_pdf_attachment->set_extensions(array('pdf','PDF'));

Hinweis zu Dateigrößen:
Für den Upload gelten dieselben Dateigrößenbeschränkungen wie beim Upload von Artikelbildern.
Standardmäßig ist die unter Konfiguration > Maximale Werte > Größe für Dateiupload auf 2 MB gesetzt (2048000).
Sollen größere pdfs verwendet werden, diesen Wert nach Wunsch erhöhen.

DEINSTALLATION

1.
Die Datei uninstall.sql im Ordner SQL mit einem Texteditor öffen und den Inhalt kopieren

2.
Zen Cart Administration öffnen und unter Tools > SQL Patches installieren den Inhalt ins Feld hineinkopieren und absenden.

3.
Alle von diesem Modul neu angelegten Dateien vom Server löschen.
Alle von diesem Modul geänderten Dateien wieder mit den Originalversionen bzw. den Dateien Ihres Backups ersetzen.
